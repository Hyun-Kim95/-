# 정보처리 기능사

#### 현재 디지털 컴퓨터의 모체: 배비지의 해석 기관(계산기)

#### 컴퓨터의 세대별 특징

> Tube(진공관) -> TR(트렌지스터) -> IC(집적회로) -> LSI(고밀도집적회로) -> VLSI(초고밀도집적회로)

* 1세대: 진공관, 주기억 장치로는 자기 드럼이 이용됨, 연산속도는 ms
* 2세대: 트렌지스터를 주요 소자로 사용, 연산속도는 ㎲
* 3세대: 집적 회로, 연산속도는 ns
* 4세대: 고밀도 집적 회로가 이용, 연산 속도는 ps
* 5세대: 초고밀도 집적 회로, 전문가 시스템, 퍼지(Fuzzy) 이론, 의사 결정 지원 시스템, 패턴 인식, 연산속도는 fs

#### 아날로그 컴퓨터

* 측정기용 컴퓨터
* 연속적인 데이터 형태로 입출력이 이루어짐
* 주요 회로로 증폭 회로를 사용
* 디지털 컴퓨터보다 속도가 빠르며 전용 컴퓨터에 사용됨
* 정밀도는 0.01% 정도로 제한됨

#### 디지털 컴퓨터: 이산적인 데이터를 취급하며 논리 회로로 구성

#### 하이브리드 컴퓨터: 디지털 컴퓨터와 아날로그 컴퓨터의 혼합

##### 비트(Bit): 정보의 최소 단위로 0 또는 1로 표현

##### 바이트(Byte): 8Bit로 구성된 문자 표현 단위

##### 레코드(Record): 데이터 처리의 기본 단위

##### 블록(Block): 주기억 장치와 입출력 장치 사이에 이동되는 데이터 단위

#### ENIAC(1946년): 프로그램 외장 방식을 사용한 세계 최초의 전자계산기

* 나머지는 내장 방식으로 생각하면 됨

#### 자기 디스크의 기억 용량: 트랙수 x 섹터 수 x 섹터당 바이트 수

#### 액세스 암(Access Arm): R/W Head 를 데이터가 있는 트랙까지 이동시켜 주는 장치

#### 기억 장치별 속도 순위(빠른순 -> 느린 순)

> CPU -> Cache -> RAM -> 자기 코어 -> 자기 드럼 -> 자기 디스크 -> 자기 테이프(순차 처리만 가능)

#### 디스크

* FAT(File Allocation Table): 디스크에 저장되어 있는 각각의 파일들에 대한 정보를 나타내는 테이블
* 포맷(Format): 디스크의 모든 자료를 삭제하고 트랙과 섹터를 나누는 작업
* 디스크 정리: 윈도우즈에서 불필요한 파일들을 삭제하여 디스크의 사용 가능한 공간을 늘리는 작업
* 디스크 검사: 디스크의 오류를 검사하고 수정하는 기능

#### CPU가 명령어를 수행하는데 필요한 동작

> Fetch, Decode, Execute, Interrupt 등

#### 명령어 인출 순서

* 프로그램 카운터의 값을 번지 레지스터에 옮김
* 명령어를 주기억 장치로부터 인출
* 프로그램 카운터를 증가시킴
* 명령 코드를 명령 레지스터에 옮김

#### 제어 장치에서의 명령 사이클

* 인출 사이클(Fetch Cycle): 하나의 명령어 수행을 위해 기억 장치로부터 하나의 명령을 CPU 내의 명령 레지스터로 가지고 오는 사이클
* 간접 사이클(Indirect Cycle): 명령어를 가져오면 피연산자를 옮겨 와야 되는데 간접 주소 지정이 허용되는 경우에는 실행 사이클에 앞서 간접 사이클이 진행됨
* 실행 사이클(Execute Cycle): 인출된 명령어를 이용하여 직접 명령을 실행하는 사이클
* 인터럽트 사이클(Interrupt Cycle): 인터럽트가 발생했을 때 처리하는 사이클

#### 마이크로 프로세서(Microprocessor)

* 마이크로 컴퓨터의 CPU로 이용
* 산술 논리 연산과 제어의 능력을 갖는 집적 회로
* 가전제품, 비디오 게임기, 공장의 자동화된 기계 등의 처리 장치로 많이 쓰임
* 8비트, 16비트, 32비트, 64비트 등으로 성능과 규모가 매우 다양한 여러 제품이 생산되고 있음
* MIPS(Million Instruction Per Second): 마이크로 프로세서의 성능을 나타냄, 1초 동안 1백만 개의 연산을 수행하는 것

* 일반적인 디지털 회로
* 입력과 과거의 출력까지 반영되어 출력
* 플립플롭으로 구성된 기억 장치

#### AND: 특정 비트 또는 특정 문자를 삭제하기 위해 사용하는 연산

#### OR: 필요한 문자를 추가하는 기능

#### 조합 논리 회로

* 다른 성질의 회로와 결합하여 복잡한 형태의 회로를 구성

#### 이항(Binary) 연산: 두 개의 입력과 하나의 출력

> 사칙연산, AND(논리곱), OR(논리합) 등

#### 단항(Unary) 연산: 하나의 입력과 하나의 출력

> Shift(밀치기) 연산, Complement(보수) 연산, Move(이동) 연산, Rotate(되돌리기) 연산 등

#### 중앙 처리 장치: 제어 장치, 연산장치, 주기억 장치

#### 주기억 장치: 프로그램과 데이터를 저장하기 위한 장치

* ROM(Read Only Memory): 읽기만 가능한 메모리, 펌웨어
* EPROM: 펌웨어의 한 종류로, 자외선으로 지울 수 있음
* RAM(Random Access Memory): 일반 주기억 장치, 저장된 위치에 관계없이 찾는 시간이 동일함, 휘발성 기억 장치
  * DRAM: 집적도가 높아 PC의 메모리로 이용, 일반적인 주기억 장치, 일정 시간이 지나면 내용이 사라짐
  * SRAM: DRAM에 비해 고가, DRAM보다 속도가 빨라 캐시(Cache) 메모리로 이용, 재충전이 필요하지 않음, 전원이 공급되는 한 내용이 사라지지 않음

##### 펌웨어(Firmware): 하드웨어와 소프트웨어의 중간적인 형태, 소프트웨어를 롬(ROM)화한 것, ROM-Software 라고도 함

##### 기억 장치의 대역폭(Bandwidth): 주기억 장치가 초 당 처리할 수 있는 비트 수를 의미

#### 인스트럭션의 설계

> 연산자의 종류, 주소 지정 방식, 해당 컴퓨터 시스템 단어(Word)의 크기(비트 수)

#### 연산 장치: 산술 및 논리 연산 장치 라고도 하며 사칙 연산과 비교, 판단 등의 연산을 수행

> 논리 회로, 누산기, 가산기, 보수기
>
> 누산기: 산술 및 논리 연산의 결과를 일시적으로 기억하는 기억 장치

#### 제어 장치: 프로그램 명령어를 해석하고, 연산장치, 주기억 장치, 입출력 장치 등에게 동작을 지시

#### 보조 기억 장치: 주변 장치로, 처리 결과 등 여러 가지 데이터를 장기적으로 보관하는 장치

#### 휘발성 메모리: 램(RAM), 캐시(Cache) 등의 반도체 메모리

#### 비휘발성 메모리: 마그네틱 메모리, 롬(ROM)

#### 캐시 기억 장치: 중앙 처리 장치와 주기억 장치 사이에서 사용되는 고속 버퍼 메모리

* 중앙 처리 장치와 주기억 장치의 속도차를 줄임
* 캐시의 용량보다 큰 프로그램을 수행할 때에는 적중률이 감소됨
* 캐시와 주기억 장치의 정보 교환 단위는 블록

#### 기억 장치의 배치 전략

* 최초 적합(First Fit): 기억할 수 있는 공간 중 제일 먼저 발견된 공간에 할당
* 최적 적합(Best Fit): 기억할 수 있는 공간 중 가장 알맞은 공간에 할당
* 최악 적합(Worst Fit): 기억할 수 있는 공간 중 가장 큰 공간(차이가 많은)에 할당

#### 채널(Channel): 중앙 처리 장치의 간섭 없이 입출력 동작을 수행하도록 지시하고, 작업이 끝나면 중앙 처리 장치에게 인터럽트로 알림

* 셀렉터 채널(Selector Channel): 하나의 고속 입출력 장치를 연결
* 멀티플렉서 채널(Multiplexer Channel): 다수의 저속 입출력 기기 사이에서 정보 전송 제어

##### 8bit 컴퓨터에서 부호와 절대치 방식으로 수치 자료를 표현할 때, 첫 자리의 수가 1이면 음수, 0이면 양수를 표현함

> ex) 10001011 -> -11

##### 인코더(Encoder): 특정 값을 2진수로 변환하거나 특정 장치로부터 보내오는 신호를 여러 개의 2진 신호로 바꾸어 변환 시키는 장치

##### 디코더(=해독기): n개의 입력 신호를 2^n개의 출력선 중 하나로 선택 출력하는 회로

##### 멀티플렉서(Multiplexer): 2^n개의 입력을 받아 하나의 출력선으로 출력하는 논리회로, 입력된 여러 개 중 하나만 선택하여 2진 정보를 출력선으로 옮김

#### 가상 기억 장치(Virtual Memory): 주기억 장치의 용량이 부족하여 보조 기억 장치의 일부를 마치 주기억 장치인 것처럼 이용하는 메모리 관리 기법

#### 부동 소수점 형식: 부호, 지수부, 가수부로 구성되며 소수점은 묵시적으로 지수부와 가수부 사이에 있는 것으로 간주함

#### ALU(Arithmetic Logic Unit): 산술 논리 연산 장치

* ALU의 구성 요소: 가산기, 누산기, 데이터 레지스터, 보수기, 상태 레지스터 등
  * 데이터 레지스터: 연산에 사용되는 데이터를 일시적으로 기억

#### 전가산기(Full Adder): 두 개의 반가산기와 한 개의 OR 회로로 구성, 두 비트와 전 상태의 자리올림수를 더해서 합(S)과 최종 자리올림수(C)를 발생하는 회로

#### 반가산기: 한 개의 XOR 회로와 AND 회로로 구성

#### 제어 장치에 사용되는 레지스터: 프로그램 카운터(PC), 명령어 레지스터(IR), 명령 해독기, 기억 번지 레지스터(MAR), 기억 버퍼 레지스터(MBR) 등

##### 레지스터(Register): 중앙 처리 장치가 자료를 처리하는 동안 사용할 값이나 연산의 중간 결과를 일시적으로 저장해 두기 위해 사용하는 중앙 처리 장치 내의 고속 임시 기억 장치로서, 새로운 자료를 저장하면 기존의 자료는 소멸됨

##### 범용 레지스터: 다목적으로 사용되는 레지스터

##### 기억 레지스터: 주기억 장치 내에 있는 자료의 출입을 관장

##### 명령 레지스터: 현재 수행 중인 명령을 기억

##### 번지 레지스터: 주기억 장치의 번지를 기억

##### 프로그램 카운터: 다음에 수행할 명령어의 번지를 기억

#### PSW(Program Status Word): CPU의 작동을 제어하기 위한 기본적인 제어 정보 포함

> * CPU 내의 레지스터 가운데 산술 연산 후의 오버플로, 캐리, 부호, 인터럽트 살생을 허용하는 비트를 포함하는 레지스터
>
> * 프로그램의 실행에 필요한 정보 및 시스템의 상태를 의미

#### 시프트 레지스터

* 클록 펄스(Clock Pulse)에 의해서 기억 내용을 한 자리 씩 이동하는 레지스터

#### 명령어와 종류

* 산술, 논리, 시프트 명령: ADD, INC, CLA, SHR, SHL
* 프로그램 제어 명령: BR, JMP, SKP, CALL, RET
* 입출력 명령: IN, OUT
* 상황 설정을 위한 정보 상태를 검사하는 명령: BUN, BSA, ISZ, SKIP
* 정지 명령: HLT

#### 색인 주소 지정(Indexed Addressing): 인덱스 레지스터와 주소 부분을 더해서 주소를 지정하는 방식, 순차적인 주소 지정 등에 유리, 주소 지정에 레지스터 2 개가 사용됨

##### 릴(Reel): 자기 테이프 또는 종이 테이프 등과 같은 기록 매체를 감는 장치

##### 헤드(Head): 데이터를 읽거나 기록할 때 이용하는 장치

##### 트렉(Track): 읽기/쓰기 헤드가 움직이는 통로, 디스크 표면에서 헤드의 서로 다른 회전 반경의 위치와 그 수가 같음

#### SASD(Sequential Access Storage Device): 순차적으로 자료 접근을 하는 방식

> ex) 자기 테이프

#### 0-주소 형식: 번지 지정이 필요 없는 스택(Stack) 구조에서 연산이 이루어짐

#### 1-주소: 하나의 누산기를 가지며 모든 데이터 처리가 누산기에 의해 이루어짐

#### 2-주소: 일반적인 형식이지만, 원래의 데이터 값이 소멸됨

#### 3-주소: 연산 후 원래 데이터 값 보존 가능, 프로그램의 길이를 짧게 할 수 있지만 명령어의 길이가 길어지는 단점이 있음

#### 스택

* LIFO(Last In First Out) 구조
* 입력과 출력이 한 쪽에서만  일어남
* 0-주소 형식에서 이용
* 삽입을 Push Down, 삭제를 Pop Up 이라 함

#### 즉시 주소 지정: 오퍼랜드에 실제 데이터 저장, 주기억 장치의 참조가 없음, 오퍼랜드의 길이에 제약이 있음

#### 직접 주소 지정: 오퍼랜드에 있는 값이 실제 데이터가 있는 주소를 나타냄

#### 간접 주소 지정: 오퍼랜드에 있는 값이 실제 데이터가 있는 번지를 알려주는 데이터의 번지를 나타냄

#### 상대 주소 지정: 오퍼랜드에 있는 값에  프로그램 카운터의 값을 더해 실제 데이터가 있는 번지를 결정

#### 묵시적 주소 지정: 데이터의 위치가 별도로 지정되어 있지 않음

#### 절대 번지(Absolute Address): 기억 장치 고유의 번지로서 16진수를 이용하여 기억 장치 중의 기억 장소를 처음부터 1바이트 마다 순서대로 정해 놓은 번지

#### 상대 번지(Relative Address): 특정 번지를 기준으로 하여 상대적 위치를 나타내는 번지

#### 콘솔

* 컴퓨터의 동작 개시와 정지를 함
* 오퍼레이터는 콘솔을 통해서 프로그램뿐만 아니라 주변 장치까지도 총괄
* 입출력 장치를 선택
* 기억 장치 내 정보의 입출력과 컴퓨터와 사용자 사이에서 메시지로 의사 전달을 주고 받음

#### T플립플롭: 토글(Toggle) 플립플롭으로서, 입력이 0일 경우에는 상태가 불변이고, 입력이 1일 경우에는 보수가 출력됨, 순서 논리 회로에 해당됨

> 누를 때마다 ON, OFF 가 교차되는 스위치를 만들 때 사용됨

#### Move: 하나의 레지스터에 기억된 자료를 모두 다른 레지스터로 옮길 때 사용하는 논리 연산

#### Rotate: 밀려간 비트가 반대쪽 끝으로 다시 채워짐-순환

#### Shift: 왼쪽 또는 오른쪽으로 1Bit씩 자리를 이동시킴-자리 이동

#### Complement: 입력에 대한 1의 보수를 구함-보수

#### 프레젠테이션: 기업체의 발표회나 각종 회의 등에서 빔 프로젝터 등을 이용하여 제품에 대한 소개나 회의 내용을 요약 정리하여 많은 사람에게 효과적으로 전달하기 위한 도구

* 개체: 프레젠테이션의 한 화면을 구성하는 개개의 요소

* 확장자
  * POT: 파워포인트에서 디자인 서식 파일
  * PPT: 파워포인트의 기본 확장자
  * PPS: 슬라이드 쇼 파일의 확장자

#### 프레젠테이션의 매체

#### 데이터베이스의 목적

> 데이터의 중복성 배제, 데이터의 공유, 데이터의 독립성, 데이터의 무결성, 데이터의 보안성, 데이터의 일관성

#### 데이터베이스 디자인 단계의 순서: 데이터베이스의 목적을 정의 -> 데이터베이스에서 필요한 테이블을 정의 -> 테이블 상의 필드 정의 -> 테이블 간의 관계 정의

#### 데이터베이스 모델

* 망 데이터베이스 모델: 레코드 간의 관계가 링크로 표현, 구조는 순환을 허용하는 그래프로 표시
* 계층적 데이터베이스 모델: 트리 구조로서 레코드 간의 관계는 링크로 표현
* 관계 데이터베이스 모델: 데이터와 데이터 간의 관계는 테이블로 표현되며, 테이블의 집합으로 데이터베이스를 구현하는 모델

#### 필드의 데이터 형식 종류

> 문자열, 숫자, 통화, 날짜/시간, 예/아니오, 메모, 일련번호, 조회 마법사, OLE 개체, 하이퍼 링크

####  ORDER BY 절: SQL의 SELECT 문에서 특정 열의 값을 기준으로 정렬할 때 사용

>
> * ASC: 오름차순
> * DESC: 내림차순

#### GROUP BY절: 그룹을 만들어 실행하는 질의

> HAVING: GROUP BY절에서 각 그룹에 대한 검색 조건의 기술

#### WHERE절: 질의 결과값으로 표시될 튜플의 조건식을 기술

#### SQL 언어: 데이터 정의 언어(DDL), 데이터 조작 언어(DML), 데이터 제어 언어(DCL)

##### 데이터 정의 언어: 테이블, 필드, 인덱스, 저장 방법 등의 데이터베이스 속성을 정의하는 언어

#### SQL문

* 데이터 정의 언어
  * CREATE : 테이블, 뷰 등의 작성
  * DROP: 테이블, 뷰 등의 삭제
    * CASCADE: 제거 대상의 제거와 함께 이를 참조하는 다른 데이터 객체에 대해서도 제거 작업이 실시됨

  * INSERT: 삽입
    * INSERT - INTO - VALUES

    * UPDATE - SET - WHERE

    * DELETE - FROM - WHERE
  * SELECT: 검색
    * SELECT - FROM - WHERE

#### 테이블의 관계

* 일 대 일 관계: 두 테이블은 관련된 두 필드가 모두 기본키 이거나 고유한 인덱스이다
* 일 대 다 관계: 두 테이블은 관련된 두 필드 중 한쪽만 기본키 이거나 고유 인덱스이다
* 다 대 다 관계: 두 테이블은 두 개의 기본키와 제 3의 테이블에 2 개의 외래키를 갖게 되며, 두 개의 일 대 다 관계가 모여 다 대 다 관계를 만든다.

#### 기본키: 테이블 내에서 모든 레코드를 유일하게 식별할 수 있는 필드 또는 필드의 집합

#### 외래키: 한 테이블의 속성들의 집합

#### 인덱스: 검색, 정렬을 빠르게 실행시켜 주기 위한 설정

#### 테이블(Table): 데이터베이스의 기본 구성 요소로 측정 항목에 대한 데이터의 집합으로 행과 열로 구성됨

#### 보고서(Report): 테이블의 자료를 출력하기 위하여 만드는 출력 양식으로 대량의 데이터를 요약하고 그룹화 하여 각종 통계 계산이 가능함

#### 질의(Query): 데이터베이스 내에 저장되어 있는 데이터를 관리하는 명령어의 집합

#### 폼(Form): 액세스(Access)에서 그래픽 입출력 틀 화면

#### 차수(Degree): 관계형 데이터 모델에서 속성의 개수를 의미함

#### 카디널리티(Cardinality): 튜플의 갯수

#### 도메인(Domain): 해당 속성에서 취할 수 있는 값

* 고급 필터: 조건식을 워크시트에 기술하고 조건에 맞는 데이터를 추출하는 기능
* 피벗 테이블: 필요한 데이터 필드를 행, 열, 쪽, 데이터 등에 배치하고 데이터에 함수를 적용하여 표시하는 집계 기능
* 부분합: 입력되어 있는 데이터를 종류별로 쉽게 계산해 주는 기능, 이 기능을 사용 전에 반드시 정렬을 수행해야 함, 그룹 윤곽 보기를 할 수 있음

#### 응답 시간(Turnaround Time): 시스템에서 결과가 얻어질 때까지의 시간

#### 서치 시간(Search Time): 해당 데이터가 있는 섹터에 헤드를 위치시키는데 걸리는 시간

#### 엑세스 타임(Access Time): 컴퓨터의 기억 장치나 디스크 등에서 정보를 꺼내는데 걸리는 시간

#### 윈도우즈의 특징

* 선점형 멀티 태스킹(Preemptive Multi-tasking): 한 대의 시스템에서 독립된 둘 이상의 작업을 병행하여 처리하는 방법
* 그래픽 사용자 인터페이스(GUI): 아이콘과 같은 그래픽을 통해 정보를 교환하는 작업 환경
* 자동 감지 설치(PnP: Plug and Play): 새 하드웨어를 설치할 때 자동으로 처리해 주는 기능
* OLE 기능의 지원: 여러 응용 프로그램에서 작성된 문자나 그림을 자유롭게 삽입할 수 있는 기능

#### 운영체제

* 구성

#### 운영체제의 특성: 효율성, 신뢰성, 용이성

#### 커널(Kernel)

> 운영체제의 가장 핵심적인 부분으로 항상 주기억 장치에 상주, 시스템의 자원을 관리, 프로세스 관리, 입출력 관리, 파일 관리, 메모리 관리, 시스템 호출, 프로세스 간 통신을 관리

#### 클립보드: 윈도우즈 작업 도중 복사 또는 이동할 파일이 잠시 기억되는 임시 기억 장소

#### 버퍼링: 정보의 송수신을 원할하게 하기 위해서 정보를 일시적으로 저장하여 처리 속도의 차를 흡수하는 방법

#### 스풀링: 출력 정보를 보조 기억 장치에 일시적으로 저장하여 CPU의 효율성을 향상시키는 방법

> 스풀(Spool): 입출력 장치와 CPU의 속도 차이를 해결하기 위한 병렬 처리 방법

#### 폴링(Polling): 컴퓨터가 단말 장치에게 '송신할 데이터가 있는가' 라고 묻는 것

#### 인터럽트: 작업 수행 중 예기치 못한 돌발적인 사태가 발생하여 잠시 작업 수행을 멈추고 상황에 맞는 처리를 한 후, 다시 프로그램을 진행하는 것

> 인터럽트 발생시
>
> * 하드웨어적 우선순위: 데이지 체인으로 모든 장치를 우선순위에 따라 직렬로 연결하여 검사(직렬)
>
>   ​										병렬 처리 인터럽트 방식(병렬)
>
> * 소프트웨어적 우선순위: 폴링 방식으로 프로그램에 의해서 우선순위 검사

##### 인터럽트 동작 순서

> CPU에게 인터럽트 요청 -> 수행 중인 프로그램 저장 -> 인터럽트 처리 루틴 -> 인터럽트 취급 루틴 -> 프로그램으로 복귀

#### 단편화(Fragmentation): 주기억 장치에서 실제로 작업에 의해 사용되지 않고 유용하게 사용될 수 없는 부분적인 기억공간

#### 선점 스케줄링

* RR, SRT, MFQ 등

#### 비선점(Non-Preemptive) 스케줄링: 1개의 프로세스가 CPU를 사용하고 있으면 프로세스의 작업이 완료되기까지는 다른 프로세스가 CPU를 점유할 수 없는 스케줄링 기법

* FIFO, SJF, HRN, 기한부 스케줄링, 우선순위 스케줄링 등

#### FIFO(First In First Out): 주기억 장치 내에 가장 먼저 들어온, 가장 오래된 페이지를 교체할 페이지로 선택하는 기법

#### SJF(Shortest Job First) 스케줄링

* 처리하여야 할 작업 시간이 가장 적은 프로세스에게 CPU 할당
* 작업들이 시스템을 통과할 때 최소 평균 대기 시간을 제공
* 대화식 시분할 시스템에 부적합
* 비선점 스케줄링 방식

#### HRN(Highest Response-ratio Next) 스케줄링

* SJF 스케줄링의 긴 작업과 짧은 작업 간의 지나친 불평등을 보완한 스케줄링 기법

#### SRT(Shortest Remaining Time) 스케줄링

* 새로운 프로세스를 포함하여 남아 있는 작업이 완료되기까지의 추정치가 가장 적은 프로세스에게 CPU를 할당

#### 디스켓의 초기화 작업

* FORMAT A:/V 명령은 포맷시 디스크 이름을 지정할 때 사용

* 초기화 작업이 끝나면 디스켓에 트랙과 섹터가 형성되어 자료들을 기억시킬 수 있음
* FORMAT A:/S 명령은 A 드라이브의 디스켓을 초기화한 후 시스템 파일을 복사함

#### FAT(파일 할당표: File Acation Table): 파일이 저장되어 있는 클러스터(Cluster)들의 위치를 제공하는 일종의 파일 배치표

* FAT32는 대용량의 파일 시스템 사용을 지원

* 같은 디스크 용량을 비교할 때 FAT32 는 FAT16에 비해 클러스터의 크기가 커지며, 숫자는 작아짐

#### UNIX

* 명령어 해석기는 운영체제의 셸(Shell)에 포함되어 있음
* 이식성이 높음
* 커널 코드의 대부분이 C 언어로 작성되어 있음
* 커널의 크기가 비교적 작음
* CUI(Character User Interface) 환경을 제공함
* 멀티태스킹 및 멀티 유저 시스템
* 시분할 온라인 대화식 시스템
* 미니 컴퓨터 및 인터넷 서버용 컴퓨터의 운영체제로 많이 사용됨
* 계층적 트리 구조로 파일들을 관리
* 다양한 개발 도구를 지우너하여 프로그램의 개발과 유지가 용이함
* DOS와 달리 대문자, 소문자를 구분
* 시스템 관리자의 프롬프트는 "#", 일반 사용자의 프롬프트는 "$"

#### i-node테이블

#### 데이터 교환 방식

* 메시지 교환 방식
  * 이용자의 형편에 따라 우선 순위 전송이 가능
  * 고장이 난 터미널로 가는 메시지를 교환기가 보관하거나 지정된 다른 터미널로 전송 가능
  * 같은 내용의 메시지를 동시에 여러 곳의 터미널로 전송 가능
  * 메시지의 분실을 방지하기 위해 번호를 부여하거나 전송 날짜, 시간 메시지에 추가 전송이 가능

#### 정보 통신 시스템

* 데이터 전송계: 단말장치, 데이터 전송 회선, 통신 제어 장치
* 데이터 처리계: 컴퓨터

#### 정보 통신 교환망

* 회선 교환망

  > 데이터를 전송하기 전에 컴퓨터와 단말기 사이에 통신 회선(전송 회선)을 설정한 통신 정보를 송수신하는 방식을 취하는 교환망

  * Point to Point 방식으로 통신 회선을 고정적으로 할당함
  * 전송 중 항상 같은 경로를 갖게 됨
  * 고정된 대역폭 전송 방식, 코드 변환이 안됨
  * 실시간 대화용 방식의 응용 가능

* 메시지 교환망

  > 축적 후 전달 또는 전문 교환형 통신망으로, 기억 장치에 정보를 기억해 두었다가 단말기에게 전송하는 방식

* 패킷 교환망

  > 전송하고자 하는 데이터를 보관해 패킷 단위로 분해한 후 전송하는 방식

#### 전송 제어 문자

* STX: 본문의 시작 및 헤딩의 종료
* ACK: 수신된 정보 메시지에 대한 긍정 응답
* NAK: 수신된 정보 메시지에 대한 부정 응답
* SOH: 정보 메시지 헤딩의 시작
* ENQ: 상대국에 데이터 링크의 설정 및 응답을 요구

#### 아날로그 변조 방법

* 진폭 편이 변조(ASK: Amplitude Shift Keying)
* 위상 편이 변조(PSK: Phase shift Keying)
* 주파수 편이 변조(FSK: Frequency Shift Keying)
* 진폭 위상 변조(QAM: Quadrature Amplitude Modulation)

#### VAN: 통신 사업자에게 회선을 임차하여 부가 가치가 높은 서비스를 제공하는 통신망

#### LAN(Local Area Network, 근거리 통신망)

* 다수의 사용자가 자원이나 네트워크 프린터의 공유 가능
* 통합성, 신뢰성이 높고 기기 간의 상호 독립적인 상태에서 통신이 가능
* LAN의 표준
  * IEEE 802.2: 논리적 링크 제어
  * IEEE 802.3: CSMA/CD
  * IEEE 802.5: 토큰링
  * IEEE 802.10: 네트워크 보안
* LAN의 구성 요소: 브릿지, 라우터, 전송 매체 등

#### MAN(Metropolitan Area Network): 여러 개의 LAN을 포함하는, 도시를 근간으로 한 네트워크

#### 종합 정보 통신망(ISDN: Integrated Services Digital Network)

> 특징

* 하나의 회선으로 여러 서비스를 종합적으로 제공
* 디지털 통신이 가능
* 기존의 전화망에 서비스를 그대로 유지

> 채널

* A채널: 아날로그 음성 신호를 전송
* B채널: 정보용 채널로, 사용자가 보내고자 하는 데이터를 전송
* D채널: 신호용 채널로 회선 접속 신호 정보를 전송
* H채널: 정보용 채널로, 고속 팩시밀리 등과 같은 고속의 사용자 정보를 전송

#### 에러 제어 방식

* ARQ: 정지-대기 ARQ, Go-back-N ARQ, 선택적 ARQ, 적응적 ARQ

* 적응적(Adaptive) ARQ
  * 전송 효율이 가장 좋음
  * 블록의 길이를 종적으로 변경 시킬 수 있음
  * 수신측에서 오류 발생 확률을 송신측에 알려주면, 송신측은 가장 적절한 블록 크기를 결정한 후 프레임 전송
* 연속적 ARQ
  * 정지-대기(Stop-and-Wait) ARQ의 단점을 보완하기 위한 방식
* 전진 에러 수정 방식(FEC: Forward Error Correction): ARQ 방식처럼 에러 검출 후 재전송이 이루어 지지 않고 수정되는 방식
  * 해밍 코드(Hamming Code): 1비트의 에러 검출 및 교정까지 가능한 코드

#### 데이터 통신 시스템

* 단방향 통신(Simplex): 한쪽 방향으로만 정보 전송이 가능(TV, 라디오)
* 반이중 통신(Half Duplex): 양쪽 방향 통신이 가능하지만 동시에는 불가능한 형태(무전기)
* 전이중 통신(Full Duplex): 양쪽 방향으로 동시에 정보의 전송이 가능한 경우(전화)

#### HDLC(High-level Data Link Control)

* HDLC Frame(프레임)의 구성
  * 시작 플래그(Flag), 주소부(Address), 제어부(Control), 정보부(Information), FCS, 종료 플래그(Flag)

* 점 대 점 및 멀티 포인트에서 사용
* 반이중과 전이중 통신 모두 지원
* 동기식 전송 방식 사용
* 에러 제어 방식으로 Go-Back-N ARQ 사용
* 비트 위주의 프로토콜

#### CRC(Cyclic Redundancy Check)

* 에러 검사 방식 중 가장 우수함
* 여러 비트에서 발생하는 집단 에러도 검출 가능
* 블록 단위로 데이터가 전송될 때 오류를 검출, 정정할 수 있는 방식

#### 백색 잡음

* 분자나 원자의 열 운동에 의해서 생기는 잡음
* 열 잡음 또는 가우스 잡음 이라고도 함
* 모든 주파수에 걸쳐서 존재하며, 제거될 수 없는 잡음임

#### 충격성 잡음(Impulse Noise)

> 전송 시스템에서 순간적으로 일어나는 높은 진폭의 잡음으로 돌발적 잡음이라고도 하며, 주로 기계적인 충격이나 번개와 같은 외부의 전자기적 충격 또는 통신 시스템의 결함에 의해 발생

#### 데이터 통신에 의한 처리 형태

> 데이터 통신은 데이터가 입력되면 즉시 처리하는 Real Time Processing System(실시간 처리 시스템)에 적합

#### 상호 변조 잡음

> 단일 주파수 대역폭을 분할하여 사용하는 각 채널 간 중복으로 인한 잡음 오류

#### 위상 왜곡

> 재생 펄스가 여러 가지 원인에 의해 올바른 시간적 위치에서 벗어나 신호 위상이 연속적으로 변하는 현상

#### 정보 전달의 5단계: 회로연결 -> 링크 확립 -> 메시지 전달 -> 링크 절단 -> 회로절단

#### 분산 네트워크

* 지역별, 기능별로 호스트 컴퓨터가 분화되어 있는 방식
* 작업이 분산적으로 진행되기 때문에 데이터 처리 효율성은 높으나 관리가 쉽지 않음

#### 가청 주파수의 범위: 20Hz ~ 20KHz

#### 샘플링 주파수: 어떤 신호를 디지털적으로 재생하기 위한 샘플링 주파수는 샘플되는 최고 주파수에 비해 최소 2배 이상 되어야 함

#### 지연 왜곡: 주파수에 따라 전송 속도가 달라서 수신측의 신호가 일그러지는 현상

#### Multi-Programming: 하나의 처리기로 여러 개의 프로그램 수행

#### 통신 방식

* Multi-Point: 두 개 이상 다수의 터미널이 하나의 통신 회선으로 연결되는 방식으로, 폴링과 셀렉션을 사용
* Point-to-Point: 중앙의 컴퓨터와 터미널이 1:1로 연결되는 방식

#### 망형의 통신 회선의 링크 수: n(n-1)/2

#### 미디어

* 텔레텍스트(Teletext)
  * TV 전파를 활용하여 다른 신호를 겹쳐서 정보를 제공하는 서비스
  * 방송국 측에서 TV 화면과 화면 사이의 수직 귀선 시간을 이용하여 정보를 한 방향으로만 전송
  * 문자 다중 방송이라고도 함(대부분 TV 화면 하단의 자막으로 표시)
* 비디오텍스(Videotex): TV 수상기와 전화망을 이용한 형태로 다양한 서비스를 제공하는 유선 통신 뉴 미디어
* HDTV: 고품위(High Definition) TV, 기존의 TV 해상도를 비약적으로 향상시킨 고화상 텔레비전, 주사선 수 1125개(현행525개)
* CATV(CAble TeleVision): 불특정 다수에게 무선 형태로 보내는 종래의 TV 방송과는 달리 광케이블 등의 유선 케이블을 통하여 CATV 가입자에게 자체적으로 제작된 콘텐츠를 제공하는 유선 중계 방송

#### 이동 통신의 접속 방식

* 코드 분할 다원 접속 방식(CDMA)

#### 가드 밴드(Guard Band): 주파수 분할 다중화기(FDM: Frequency Division Multiplexer)에서 사용되는 채널 간 완충 영역 기능

#### 데이터 통신의 특징

* 고속 전송이 가능
* 부호 통신
* 전신 전송이 기원

#### WDM(Wavelength Division Multiplexing)

> 파장 분할 다중화 방식으로 서로 다른 파장의 여러 채널을 단일 광섬유를 통해 전송하는 방식

#### 광통신의 다이오드

* 발광 소자
  * 발광 다이오드(LED: Light Emitting Diode)
  * 레이저 다이오드(LD: Laser Diode)
* 수광 소자
  * 포토 다이오드(PD: Photo Diode)
  * 에벌런시 다이오드(APD: Averancy Photo Diode)

#### FTTH(Fiber To The Home)

> 인터넷을 사용하는 집안까지 광섬유를 설치하여 사용하는 초고속 인터넷 방식으로 FTTP(Fiber To The Premises) 라고도 함

#### 프로세스(Process)

* 실행(Running) 중인 프로그램을 의미
* 프로그램을 실행하여 처리하는 단위
* 운영체제의 PCB(Process Control Block) 내에 존재

#### 프로세스를 생성하는 데 필요한 작업

> PCB 생성, 프로세스의 우선순위 변경, 프로세스 이름 결정, 프로세스 현재 상태 파악

#### 펄스 코드 변조(PCM: Pulse Code Modulation): 아날로그 신호를 디지털 펄스로 변환하여 전송하고, 수신측에서는 이를 다시 본래의 아날로그 신호로 환원시키는 방식

#### PCM 변조 과정: 표본화 -> 양자화 -> 부호화 -> 복호화 -> 여과

#### ISO의 OSI 7 참조 모델

> 응용계층(최상위 계층) -> 표현 계층 -> 세션 계층 -> 전송 계층 -> 네트워크 계층 -> 데이터 링크 계층 -> 물리 계층(최하위 계층)

* 응용 계층: 사용자에게 특정한 서비스를 제공하는 기능을 수행
* 세션 계층: 사용자와의 인터페이스로 종점 간의 기본적인 연결 서비스에 기능을 부가하여 실체가 특성에 맞게 데이터를 교환할 수 있는 연결 서비스를 제공하고 제어 기능을 수행
* 네트워크 계층: 라우팅, 인터네트워킹과 같은 네트워크를 연결하는 데 필요한 데이터 교환기능의 제공 및 관리 규정
* 데이터 링크 계층: 두 개의 인접한 개방 시스템 간에 신뢰성 있고 효율적인 정보 전송을 할 수 있도록 함, 데이터를 일정 길이의 프레임으로 전송
* 물리 계층: 상위 계층으로부터 내려온 데이터를 비트열로 전송, 전송 매체에 전기적 신호를 전송, RS-232C와 같은 물리적 접속 기능을 제공

#### 부호 전송 방식

* 단극성(unipolar): 데이터 비트가 '0' 일 때는 (-), '1' 일 때는 (+) 전압을 유지하는 방식, 동일한 신호가 계속되는 경우에는 상태 변화가 없음

#### 시분할 멀티 플렉싱(시분할 다중화기 방식)(Time Division Multiplexer)

* 각 채널당 고정된 프레임을 구성하여 전송한다
* 시간을 타임 슬롯(Time Slot) 이라는 기본 단위로 나누고, 이들을 일정한 크기의 프레임으로 묶어서 채널별로 특정 시간대에 해당하는 슬롯에 배정하는 방식

#### 인터넷에서 제공하는 서비스

* FTP(File Transfer Protocol): 파일 전송 프로토콜 일을 송수신하기 위한 서비스
* 텔넷(Telnet): 원거리에서도 자신의 PC처럼 접속하여 사용 가능한 서비스
* 유즈넷(Usenet): 공통의 관심사에 대해 토론할 수 있는 서비스
* 전자 우편(E-mail): 인터넷 이용자 간에 편지를 주고 받게 하는 시스템
* 고퍼(Gopher): 데이터를 색인해서 정보를 메뉴 방식으로 검색할 수 있도록 만든 서비스

#### 프로토콜의 기능

> 단편화와 조립화, 캡슐화, 연결 제어, 흐름 제어, 오류 제어, 동기화, 순서 결정, 주소 기법, 다중화, 전송 서비스

#### SMTP(Simple Mail Transfer Protocol): 전자 우편을 송신하기 위한 프로토콜

#### MPEG(Moving Picture Experts Group): 동화상의 크기를 줄이는 압축 기술

#### 멀티미디어의 요소

> 그래픽, 비디오, 사운드 등

#### IPTV(Internet Protocol TeleVision): 인터넷을 통해 텔레비전 시청이 가능하게 제공해 주는 서비스
