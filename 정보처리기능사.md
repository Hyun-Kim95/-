# 정보처리 기능사

#### 현재 디지털 컴퓨터의 모체: 배비지의 해석 기관(계산기)

#### 컴퓨터의 세대별 특징

* 1세대: 진공관, 주기억 장치로는 자기 드럼이 이용됨, 연산속도는 ms
* 2세대: 트렌지스터를 주요 소자로 사용, 연산속도는 ㎲
* 3세대: 집적 회로, 연산속도는 ns
* 4세대: 고밀도 집적 회로가 이용, 연산 속도는 ps
* 5세대: 초고밀도 집적 회로, 전문가 시스템, 퍼지(Fuzzy) 이론, 의사 결정 지원 시스템, 패턴 인식, 연산속도는 fs

> 연산속도: micro second(10^-6)

#### PSW(Program Status Word): CPU의 작동을 제어하기 위한 기본적인 제어 정보 포함

> CPU 내의 레지스터 가운데 산술 연산 후의 오버플로, 캐리, 부호, 인터럽트 살생을 허용하는 비트를 포함하는 레지스터

#### 단말장치: 컴퓨터로 데이터를 송수신 하기 위해 필요한 입출력 장치

#### 입출력장치

* MICR, OMR, OCR: 턴어라운드 시스템의 입출력 매체

* 호퍼(Hopper): 천공된 카드를 읽기 위해 쌓아 놓는 곳
* POS(Point Of Sales): 백화점, 슈퍼마켓 등 유통업의 전산시스템으로 바코드에 의한 관리를 함

#### 명령어(Instruction): 명령 코드부(OP-Code) 와 주소부(Operand)로 구성

> OP-Code: 명령어의 첫 번째 바이트에 기억되며 명령어의 형식, 연산자, 자료의 종류를 나타냄
>
> Operand: 명령어의 주소 부분에 해당, 주소 또는 주소를 구하기 위해 필요한 정보와 명령의 순서를 나타냄

#### 중앙 처리 장치: 제어 장치, 연산장치, 주기억 장치

#### 채널(Channel): 중앙 처리 장치의 간섭 없이 입출력 동작을 수행하도록 지시하고, 작업이 끝나면 중앙 처리 장치에게 인터럽트로 알림

##### 8bit 컴퓨터에서 부호와 절대치 방식으로 수치 자료를 표현할 때, 첫 자리의 수가 1이면 음수, 0이면 양수를 표현함

> ex) 10001011 -> -11

##### 인코더: 특정 값을 2진수로 변환하거나 특정 장치로부터 보내오는 신호를 여러 개의 2진 신호로 바꾸어 변환 시키는 장치

##### 디코더(=해독기): n개의 입력 신호를 2^n개의 출력선 중 하나로 선택 출력하는 회로

##### 멀티플렉서: 2^n개 입력 신호 중 하나를 선택하여 출력하는 회로

#### 부동 소수점 형식: 부호, 지수부, 가수부로 구성되며 소수점은 묵시적으로 지수부와 가수부 사이에 있는 것으로 간주함

#### 연산장치: 논리 회로, 누산기, 가산기, 보수기

* 누산기: 산술 및 논리 연산의 결과를 일시적으로 기억하는 기억 장치

#### 제어 장치에 사용되는 레지스터: 프로그램 카운터(PC), 명령어 레지스터(IR), 명령 해독기, 기억 번지 레지스터(MAR), 기억 버퍼 레지스터(MBR) 등

##### 레지스터(Register): 중앙 처리 장치가 자료를 처리하는 동안 사용할 값이나 연산의 중간 결과를 일시적으로 저장해 두기 위해 사용하는 중앙 처리 장치 내의 고속 임시 기억 장치로서, 새로운 자료를 저장하면 기존의 자료는 소멸됨

##### 릴(Reel): 자기 테이프 또는 종이 테이프 등과 같은 기록 매체를 감는 장치

##### 헤드(Head): 데이터를 읽거나 기록할 때 이용하는 장치

#### SASD(Sequential Access Storage Device): 순차적으로 자료 접근을 하는 방식

> ex) 자기 테이프

#### 직접 주소 방식: 주소 부분에 있는 값이 실제 데이터가 있는 주기억 장치 내의 주소를 나타냄

#### 절대 번지(Absolute Address): 기억 장치 고유의 번지로서 16진수를 이용하여 기억 장치 중의 기억 장소를 처음부터 1바이트 마다 순서대로 정해 놓은 번지

#### 상대 번지(Relative Address): 특정 번지를 기준으로 하여 상대적 위치를 나타내는 번지

#### 콘솔

* 컴퓨터의 동작 개시와 정지를 함
* 오퍼레이터는 콘솔을 통해서 프로그램뿐만 아니라 주변 장치까지도 총괄
* 입출력 장치를 선택
* 기억 장치 내 정보의 입출력과 컴퓨터와 사용자 사이에서 메시지로 의사 전달을 주고 받음

#### T플립플롭: 토클 플립플롭으로서, 입력이 0일 경우에는 상태가 불변이고, 입력이 1일 경우에는 보수가 출력됨

> 누를 때마다 ON, OFF 가 교차되는 스위치를 만들 때 사용됨

#### Move: 하나의 레지스터에 기억된 자료를 모두 다른 레지스터로 옮길 때 사용하는 논리 연산

#### Rotate: 밀려간 비트가 반대쪽 끝으로 다시 채워짐-순환

#### Shift: 왼쪽 또는 오른쪽으로 1Bit씩 자리를 이동시킴-자리 이동

#### Complement: 입력에 대한 1의 보수를 구함-보수

#### 프레젠테이션: 기업체의 발표회나 각종 회의 등에서 빔 프로젝터 등을 이용하여 제품에 대한 소개나 회의 내용을 요약 정리하여 많은 사람에게 효과적으로 전달하기 위한 도구

* 개요: 프레젠테이션에서 프레젠테이션의 흐름을 기획
* 개체: 프레젠테이션의 한 화면을 구성하는 개개의 요소

#### 데이터 교환 방식

* 메시지 교환 방식
  * 이용자의 형편에 따라 우선 순위 전송이 가능
  * 고장이 난 터미널로 가는 메시지를 교환기가 보관하거나 지정된 다른 터미널로 전송 가능
  * 같은 내용의 메시지를 동시에 여러 곳의 터미널로 전송 가능
  * 메시지의 분실을 방지하기 위해 번호를 부여하거나 전송 날짜, 시간 메시지에 추가 전송이 가능

#### 정보 통신 교환망

* 회선 교환망

  > 데이터를 전송하기 전에 컴퓨터와 단말기 사이에 통신 회선(전송 회선)을 설정한 통신 정보를 송수신하는 방식을 취하는 교환망

* 메시지 교환망

  > 축적 후 전달 또는 전문 교환형 통신망으로, 기억 장치에 정보를 기억해 두었다가 단말기에게 전송하는 방식

* 패킷 교환망

  > 전송하고자 하는 데이터를 보관해 패킷 단위로 분해한 후 전송하는 방식

#### 데이터 통신에 의한 처리 형태

> 데이터 통신은 데이터가 입력되면 즉시 처리하는 Real Time Processing System(실시간 처리 시스템)에 적합

#### ISO의 OSI 7 참조 모델

> 응용계층(최상위 계층) -> 표현 계층 -> 세션 계층 -> 전송 계층 -> 네트워크 계층 -> 데이터 링크 계층 -> 물리 계층(최하위 계층)

#### 부호 전송 방식

* 단극성(unipolar): 데이터 비트가 '0' 일 때는 (-), '1' 일 때는 (+) 전압을 유지하는 방식, 동일한 신호가 계속되는 경우에는 상태 변화가 없음

#### 시분할 멀티 플렉싱(시분할 다중화기 방식)

* 각 채널당 고정된 프레임을 구성하여 전송한다
* 시간을 타임 슬롯(Time Slot) 이라는 기본 단위로 나누고, 이들을 일정한 크기의 프레임으로 묶어서 채널별로 특정 시간대에 해당하는 슬롯에 배정하는 방식
